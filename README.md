## References

- [Nielsen Norman Group: Glassmorphism - Definition and Best Practices](https://www.nngroup.com/articles/glassmorphism/)
  - Overview of the trend, including usability considerations like contrast and
    legibility.
- [LogRocket Blog: What is glassmorphism? How to create glassmorphic designs](https://blog.logrocket.com/ux-design/what-is-glassmorphism/)
  - Summarizes the core ingredients such as translucency, blur, and layered depth.
- [Ramotion: What is Glassmorphism: Principles, Practices & Examples](https://www.ramotion.com/blog/what-is-glassmorphism/)
  - Breaks down common visual specs (opacity, borders, highlights) and use cases.
- [Clay: How Glassmorphism in UX Is Reshaping Modern Interfaces](https://clay.global/blog/glassmorphism-ui)
  - Examples and guidance on when glass-like surfaces support the UI hierarchy.
- [MDN: backdrop-filter](https://developer.mozilla.org/en-US/docs/Web/CSS/backdrop-filter)
  - Technical reference for the CSS blur effect that powers glassmorphism.

## Expansion ideas

Based on the current hero, cards, principles, and styleguide sections, the demo
could grow into a fuller UI kit by adding:

- A glassmorphism sidebar or dock with quick actions and profile status.
- Toast notifications and alert banners for layered overlays.
- A modal or command palette using the native `<dialog>` element.
- Frosted form controls (search, inputs, toggles, sliders) with focus states.
- Data widgets such as KPI tiles, charts, or activity timelines.
- A compact media player or calendar card to showcase dense layouts.
